# Plan: Bug Fixes - Navegación entre Sectores del Juego

## Problemas Identificados

### Bug #1: QuestLogView - Botón "IR AHORA" no navega al stage correcto

**Archivo:** `components/views/QuestLogView.tsx:112`
**Problema:** El handler `onOpenQuest` recibe `quest.target_stage` (un objeto Stage completo), pero el tipo `QuestEntry.target_stage` está definido como `any` en lugar de `Stage`.

**Impacto:** La navegación desde QuestLog a StageDetails puede fallar o mostrar datos incorrectos.

### Bug #2: ViewShell - Falta navegación entre capítulos en CampaignMapView

**Archivo:** `components/views/CampaignMapView.tsx`
**Problema:** Solo se muestra `currentChapterIndex = 0` (hardcodeado). No hay botones o lógica para navegar entre capítulos. Los jugadores no pueden acceder a otros capítulos de la campaña.

**Impacto:** Los jugadores están "atascados" en el primer capítulo, sectores posteriores del juego son inaccesibles.

### Bug #3: GlobalNavigation - Vistas sin acceso directo

**Archivo:** `components/layout/GlobalNavigation.tsx` (asumiendo existe)
**Problema:** La navegación inferior solo muestra algunas vistas. Vistas como 'guild', 'tower', 'arena', 'quests' pueden no estar accesibles desde allí, forzando al usuario a usar atajos de teclado o volver al home.

### Bug #4: RPGHomeView - Falta botón de acceso a QuestLog

**Archivo:** `components/views/RPGHomeView.tsx:154-178`
**Problema:** Los QuickActions solo muestran 'guild' y 'tower', pero no hay forma de navegar a 'quests' desde la home. El objetivo actual apunta a 'campaign' pero no hay acceso a 'quests'.

**Impacto:** Los jugadores no descubren la funcionalidad de QuestLog a menos que usen el atajo de teclado 'Q'.

### Bug #5: StageDetailsView - Falta botón de regreso a Campaign

**Archivo:** `components/views/StageDetailsView.tsx`
**Problema:** El botón de regreso solo está disponible en `onBack`, pero no hay forma de navegar de vuelta a 'campaign' desde esta vista usando la navegación principal.

### Bug #6: BattleScreenView - Navegación post-batalla

**Archivo:** `components/views/BattleScreenView.tsx`
**Problema:** Después de la batalla, el resultado llama a `onBack()` que va a 'campaign'. Si el jugador estaba en 'quests' y luego en 'stage_details', la navegación rompe el flujo esperado.

## Solución Propuesta

### Fix #1: QuestLogView

```typescript
// Cambiar el tipo de target_stage
interface QuestEntry {
  // ...otros campos
  target_stage: Stage; // En vez de any
}
```

### Fix #2: CampaignMapView - Navegación entre capítulos

```typescript
// Agregar funcionalidad de navegación entre capítulos
const [currentChapterIndex, setCurrentChapterIndex] = useState(0);

// Agregar botones de anterior/siguiente capítulo
<Button
  onClick={() => setCurrentChapterIndex(i => Math.max(0, i - 1))}
  disabled={currentChapterIndex === 0}
>
  Capítulo Anterior
</Button>
<Button
  onClick={() => setCurrentChapterIndex(i => Math.min(chapters.length - 1, i + 1))}
  disabled={currentChapterIndex === chapters.length - 1}
>
  Siguiente Capítulo
</Button>
```
